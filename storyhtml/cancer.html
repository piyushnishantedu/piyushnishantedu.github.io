Business Objectives and constraints

  • No low-latency requirement.
  • Interpretability is important.
  • Errors can be very costly.
  • Probability of a data-point belonging to each class is needed.

Metric(s):

  • Multi class log-loss
  • Confusion matrix

Machine Learing Objectives and Constraints

Objective: Predict the probability of each data-point belonging to each of the nine classes.


Constraints:


  • Interpretability
  • Class probabilities are needed.
  • Penalize the errors in class probabilites => Metric is Log-loss.
  • No Latency constraints.

Split the training data set into three parts train, validation and test set. Validation set is used for hyper parameter tuning and test set is completely unknown for the trained model. We split the data set in 64%, 16% and 20%. 64% of total data is used for traininng, 16% data used for testing and 20% data for validations

Data descriptions

  • Traing variant is comma seperated file having unique id, gene, variation and class columns
  • ID : the id of the row used to link the mutation to the clinical evidence
  • Gene : the gene where this genetic mutation is located
  • Variation : the aminoacid change for this mutations
  • Class : 1-9 the class this genetic mutation has been classified on
  • training_text file having two columns one is uniqu id and other is text

Steps

  • merge two files on the basis of unique id column valus.
  • create a single file which has the fetures from both files.

Step 1: Preprocessing

  • Remove all special characters from text data and replace it with space.
  • replace multiple white spaces with one white space
  • normalize text by lowering the characters in the word or sentences.
  • Remove all the stop words from text data

Step-2: Data cleaning

  • After merging two files there are many rows havinmg text data column value are NAN.
  • replace NAN value from text column by concatenating gene and variation columns

Data split

  • validation data set used for bias variance tradeoff to tune hyper parameter
